# Contributing to RevenueCat.NET

Thank you for your interest in contributing to RevenueCat.NET!

## Development Setup

### Prerequisites
- .NET 8 SDK or higher
- Your favorite IDE (Visual Studio 2022, Rider, or VS Code)

### Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch
4. Make your changes
5. Run tests
6. Submit a pull request

### Building

```bash
dotnet build RevenueCat.NET.sln
```

### Running Tests

```bash
dotnet test RevenueCat.NET.sln
```

### Code Style

This project follows standard C# coding conventions:
- Use file-scoped namespaces
- Use primary constructors where appropriate
- Use records for immutable data models
- Enable nullable reference types
- Follow SOLID principles
- Write clean, self-documenting code
- Add XML documentation for public APIs

### Commit Messages

- Use clear and descriptive commit messages
- Start with a verb in present tense (Add, Fix, Update, Remove)
- Reference issue numbers when applicable

### Pull Request Process

1. Update the README.md with details of changes if needed
2. Update the CHANGELOG.md following Keep a Changelog format
3. Ensure all tests pass
4. Request review from maintainers

## Code of Conduct

Be respectful and professional in all interactions.

## Questions?

Feel free to open an issue for any questions or concerns.
